# Kid Activity Lab Content Model

One domain can hold multiple page types as long as the promise stays consistent: practical kids activities parents can run at home.

## Primary Content

Original pages are the core asset.

Examples:

- Field-test packs.
- Parent-tested activity notes.
- Original photos, clips, or diagrams.
- Activity pages improved with real observations.

These pages should lead navigation, homepage modules, and internal links.

## Utility Content

Activity cards are the reusable action layer.

They should stay short enough to run from one screen, but improve over time with:

- Parent setup notes.
- Easier and harder versions.
- Safety notes from actual materials.
- Original visuals when possible.

## Support Content

Video curation can stay on the same domain, but it should support original work instead of replacing it.

Use video pages for:

- Idea discovery.
- Source review.
- Comparing setup styles.
- Finding activities worth testing.

Avoid making curation pages the main ranking bet unless the page adds substantial original utility.

## Publishing Rule

Use curation to find ideas. Use testing to make pages worth ranking.
